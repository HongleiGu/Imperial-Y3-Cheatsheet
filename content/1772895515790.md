# NLP Cheatsheet

  

# 1 Word Representations

## One-Hot Encoding

Each token = vector of length $|vocab|$ with a single $1$ at its index. Shape: $(seq\_len,vocab\_size)$

  

**Flaws:** (1) All words equidistant — no similarity. (2) Sparse & high-dimensional. (3) Fixed vocab — OOV needs `UNK`.

  

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/one-hot.png" alt="one-hot" style="width:150px;">

  

## Word Embeddings (Word2Vec)

  

The embedding matrix $W \in\mathbb{R}^{|V|\times d}$ stores the vector for each word.  

When a **one-hot vector** is multiplied by W, it simply **selects the row corresponding to that word**.

  

_Distributional hypothesis:_ words with similar meanings appear in similar contexts. Dense, learned vectors where similar words are close together.

  

**CBOW:** predict target word from context -> $w_{t-2}, w_{t-1}, w_{t+1}, w_{t+2}$ -> $w_t$

  

**Skip-gram:** predict context from target -> $w_t$ -> $w_{t-2}, w_{t-1}, w_{t+1}, w_{t+2}$

  

CBOW and Skip-gram use the same model architecture

```

     Input       ->   Linear   ->  Embedding vector

(one-hot, size V)   (W: V × d)      (vector of d)

  

                -> Linear layer    -> Softmax

                    (W: V × d)

```

  

window size n in Word2Vec is the n words before and after a certain word or token. so in Skip-gram this meant we use the n words before and after to predict the center word, and the opposite for CBOW

  

CBOW and skip-gram share the same basic architecture, but differ in their training objective and input structure.

  
  

**Negative Sampling:** softmax over 100k+ vocab is expensive. Instead, train binary classifiers — real context word vs. k random noise words. Use sigmoid, not softmax. k = 2–5 (large data), 5–20 (small data).

  

for each training step, the model is asked "is this word a real context word (1) or a random noise word (0)?", and sigmoid outputs this probability. This replaces the single expensive softmax over the full vocab.

  

## Vector Similarity

  

**Euclidean:** $d = \sqrt{\sum(q_i - d_i)^2}$ — straight-line distance

  

**Cosine:** $\cos(\theta) = \frac{p_1 \cdot p_2}{|p_1||p_2|}$ — angle between vectors (preferred in NLP; if normalised, = dot product)

  

> **Analogy recovery:** `king − man + woman ≈ queen` (vector offsets encode relationships)

  

Cosine similarity ignores vector magnitude and compares only direction.

This is useful because frequency effects can change magnitude but not meaning.

  

## Limitations of Static Embeddings

  

- Needs large data; poor on rare/unseen words

- Antonyms cluster together ("good" ≈ "bad") because the words occur in similar contexts (e.g., "not good", "not bad") — distributional similarity conflates them.

- No morphology: "car" ≠ "cars"

- One vector per word — no sense disambiguation

  

## Byte Pair Encoding (BPE)

  

Sub-word tokenisation — solves OOV without a huge vocab.

  

**Train:** start with characters -> repeatedly merge most frequent bigram -> add to vocab. End-of-word marker `_` distinguishes suffixes and marks word boundaries. The end-of-word marker may vary by implementation, but this helps the model distinguish the 'est' as a standalone word from a suffix.

  

**Inference:** split new word into chars -> apply learned merges in order until none apply.

  

_Example:_ "lowest" -> `low` + `est_`

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/BPE2.png" alt="BPE2" width="150">

  

## TF-IDF

  

Downweights common words ("the", "a") in bag-of-words vectors.

  

$$\text{TF-IDF}(w,d) = \text{TF}(w,d) \times \log\frac{|D|}{|\{d \in D : w \in d\}|}$$

  

The denominator $|\{d \in D : w \in d\}|$ is the **document frequency** — the number of documents containing word $w$. The fraction $\frac{|D|}{df}$ is the **inverse document frequency (IDF)**. High IDF = word is rare across documents = more informative.

  

## Pre-trained Embeddings

  

|Model|Type|Notes|

|-|-|-|

|GloVe|Static|50–300d, up to 2.2M vocab; uses co‑occurrence counts|

|FastText|Static|Sub-word aware, 157 languages; handles OOV via character n‑grams|

|ELMo / BERT|Contextual|Different vector per context|

  

# 2 Classification

  

**Task:** Predict which class an input belongs to. Binary (sigmoid + 0.5 threshold) or multi-class (softmax).

  

## Naive Bayes

  

Generative model. Applies Bayes' rule with the **conditional independence assumption** (features are independent given the class).

  

$$\hat{y} = \underset{c}{\arg\max}\ P(c) \prod_{i} P(x_i | c)$$

  

**Training:** count word frequencies per class from corpus (BoW features).

**Add-one smoothing:** prevents zero probabilities for unseen words — add 1 to all counts, normalise by $|V|$.

  

**Improvements:** (1) _Binary NB_ — clip word counts to 1 per doc. (2) _Negation handling_ — prepend `NOT_` to words after negation until next punctuation.

  

**Problems:** all features equally weighted; independence assumption; ignores word order/context; can't handle unseen words.

  

In practice, the product $\prod_i P(x_i|c)$ is computed as a **sum of log probabilities** to avoid numerical underflow. So $\hat{y} = \arg\max_c \left[\log P(c) + \sum_i \log P(x_i|c)\right]$.

  

## Logistic Regression

  

Discriminative model. Directly learns $P(y|x)$.

  

$$P(y=1|x) = \sigma(w \cdot x + b), \quad \sigma(z) = \frac{1}{1+e^{-z}}$$

  

- Weights $w$ encode feature importance (learned via gradient descent)

- Loss: **cross-entropy** $\mathcal{L} = -\sum P \log Q$

- Multi-class: replace sigmoid with **softmax**, learn one weight vector per class

  

**vs Naive Bayes:** LR handles correlated features better; NB trains faster and needs less data.

  

## Neural Networks

  

Linear layers + non-linear activations. Automatically learn feature representations.

  

**Input options:** one-hot (basic) or pre-trained dense embeddings (better).

  

**Document representation:** can't just concatenate (variable length) — use mean pooling, RNN, or CNN.

  

## RNNs

  

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/RNN.png" alt="RNN" width="150">

  

For sequential data. Hidden state $h_t$ carries context forward.

  

$$h_t = \tanh(W h_{t-1} + U x_t)$$

  

- $W$: H×H (hidden-to-hidden), $U$: H×E (embedding-to-hidden)

- Final hidden state $h_T$ -> output layer

- **Vanishing gradient problem:** tanh/sigmoid derivatives < 1 -> repeated multiplication of $W$ -> gradients shrink (eigenvalues < 1) for early timesteps; when eigenvalues > 1, gradients may explode (use gradient clipping). LSTMs are designed to mitigate vanishing gradients.

  

## CNNs

  

- **Filter** slides over word embeddings (width = embed dim, height = n-gram size, typically 2–5)

- Produces a feature map -> **max pooling** extracts most important feature -> fixed-size representation

- $d$ parallel filters -> $d$-dimensional output

  

**RNN vs CNN:** CNNs good for key-phrase detection; RNNs better for long-range dependencies.

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/CNN.png" alt="CNN" width="150">

  

## Evaluation

  

**Confusion matrix** -> TP, TN, FP, FN

  

$$\text{Precision} = \frac{TP}{TP+FP} \qquad \text{Recall} = \frac{TP}{TP+FN} \qquad F_1 = \frac{2 \cdot P \cdot R}{P + R}$$

  

**Accuracy** is misleading on imbalanced classes — prefer F1.

  

**Precision** = "of all I *predicted* positive, how many were correct?" (FP in denominator = false alarms).

  

**Recall** = "of all *actual* positives, how many did I catch?" (FN in denominator = misses).

  

**Macro**: Average F1 per class equally — emphasises rare classes

  

**Micro**: Pool all TP/FP/FN across classes — emphasises frequent classes

  

# 3 Language Models

  

## Language Modelling

  

A **Language Model (LM)** assigns probabilities to word sequences.

  

$$P(w_1, \ldots, w_n) = \prod_{i=1}^{n} P(w_i \mid w_1,\ldots,w_{i-1})$$

  

Common tasks: next word prediction, masked word prediction, text generation.

  

## N-gram Language Models

  

Approximate history using the last **N−1 words**.

  

|Model|Formula|

|-|-|

|Bigram|$P(w_i \mid w_{i-1})$|

|Trigram|$P(w_i \mid w_{i-2}, w_{i-1})$|

  

Note the off-by-one: an **N-gram** model uses **N−1** words of context (so a bigram uses 1 previous word, a trigram uses 2).

  

Let the tag sequence be:

  

$$  

T = t_1, t_2, ..., t_n  

$$

  

**Chain Rule**

$$

P(T) = P(t_1,t_2,...,t_n) = \prod_{i=1}^{n} P(t_i \mid t_1,...,t_{i-1})  

$$

  

**n-gram (Markov) Assumption**

  

Assume the next tag depends only on the **previous (n-1) tags**:

$$  

P(t_i \mid t_1,...,t_{i-1}) \approx P(t_i \mid t_{i-n+1},...,t_{i-1})  

$$

  

So:

  

$$  

P(T) \approx \prod_{i=1}^{N} P(t_i \mid t_{i-n+1},...,t_{i-1})  

$$

  
  

### Probability Estimation (MLE)

  

$$P(w_i \mid w_{i-1}) = \frac{count(w_{i-1}, w_i)}{count(w_{i-1})}$$

  

## Sparsity Problem

  

Many n-grams **never appear in training data**, resulting in $P = 0$. This breaks sequence probability -> **smoothing needed**.

  

## Smoothing Methods

  

### Add-1 (Laplace) Smoothing

  

$$P(w_i \mid w_{i-1}) = \frac{count(w_{i-1}, w_i) + 1}{count(w_{i-1}) + |V|}$$

  

if Add-K $$P(w_i \mid w_{i-1}) = \frac{count(w_{i-1}, w_i) + K}{count(w_{i-1}) + K|V|}$$

  

Pros: simple. Cons: assigns **too much probability to unseen events**.

  

### Back-off

  

If n-gram unseen -> use lower-order model.

  

$$P(w_i \mid w_{i-2}, w_{i-1}) \rightarrow 0.4 \times P(w_i \mid w_{i-1})$$

  

but if seen $C(w_{i-2}w_{i-1}w_i)>0$

  

$$P(w_i \mid w_{i-2}, w_{i-1}) = \frac{C(w_{i-2}w_{i-1}w_i)}{C(w_{i-2}w_{i-1})}$$

  

### Interpolation

  

Combine multiple models:

  

$$P = \lambda_1 P_{trigram} + \lambda_2 P_{bigram} + \lambda_3 P_{unigram}, \quad \lambda_1+\lambda_2+\lambda_3=1$$

  

The $\lambda$ weights are learned on a **held-out (validation) set**, not the training set.

  

## Sentence Probability

  

Example: `<s> I want chinese food </s>`

  

$$P(I|\langle s\rangle) \cdot P(want|I) \cdot P(chinese|want) \cdot P(food|chinese) \cdot P(\langle/s\rangle|food)$$

  

## Perplexity (PPL)

  

Measures **how well a LM predicts test data**.

  

$$PPL = \left(\frac{1}{P(w_1,\ldots,w_n)}\right)^{1/n}$$

  

Lower = better. Best possible: $PPL = 1$. Uniform vocab model: $PPL = |V|$.

  

Intuitively, PPL is the **average branching factor** — how many equally likely words the model is "choosing between" at each step. PPL = 100 means the model is as confused as if uniformly choosing among 100 words at every position.

  

## Cross-Entropy vs Perplexity

  

$$H = -\frac{1}{N}\sum_{i=1}^N \log q(x_i) \qquad PPL = e^H$$

  

(or $2^H$ if log base 2)

  

## Feed-Forward Neural LM (FFLM)

  

Predict next word using **fixed context window**.

  

```

context words -> embeddings -> feed-forward NN -> softmax -> next word

```

  

Improves perplexity vs n-grams but **limited context size**.

  

## RNN Language Models

  

Process tokens sequentially.

  

$$h_t = f(x_t, h_{t-1}) \qquad P(w_{t+1} \mid h_t)$$

  

Training loss = **sum of cross-entropy across timesteps**.

  

**Teacher forcing:** use true previous token during training (prevents error accumulation). A linear + softmax head is added at each timestep.

  

## Bidirectional RNN

  

Two RNNs running in parallel:

  

```

forward: left -> right

backward: right -> left

```

  

Final representation: $[h_{forward}; h_{backward}]$. Useful for **classification tasks**.

  

Note: Bidirectional RNNs **cannot** be used for language modelling (generation), because the backward pass sees future tokens — this would be cheating. They are only suitable for tasks where the full input is available, like classification or tagging.

  

## LSTM

  

Addresses **vanishing gradients** via a cell state $c_t$ (long-term memory) and hidden state $h_t$.

  

|Gate|Purpose|

|-|-|

|Forget gate|Discard irrelevant memory|

|Input gate|Write new information|

|Output gate|Expose memory to hidden state|

  

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/LSTM.png" alt="LSTM" width="150">

  

The key insight is the **additive update** to the cell state (not multiplicative like vanilla RNN), which allows gradients to flow through many timesteps without vanishing. The gates are sigmoid-activated (output 0–1), acting as soft on/off switches.

  

## GRU

  

Simplified LSTM with an **update gate** and **reset gate**. No separate cell state — fewer parameters, faster training.

  

## Evaluation Types

  

|Type|Method|

|-|-|

|Intrinsic|Perplexity|

|Extrinsic|Downstream task performance (translation, QA, classification)|

  

# 4 Machine Translation

  

## Goal

  

**Machine Translation (MT)**: automatically translate text from a **source language** $x$ to a **target language** $y$.

  

$$y^* = \arg\max_y P(y \mid x)$$

  

## Statistical Machine Translation (SMT)

  

Using **Bayes Rule**:

  

$$P(y \mid x) = \frac{P(x \mid y)P(y)}{P(x)} \implies y^* = \arg\max_y P(x \mid y)P(y)$$

  

|Component|Formula|Role|

|-|-|-|

|Language Model|$P(y)$|Ensures fluency|

|Translation Model|$P(x \mid y)$|Explains source from target|

  

The direction $P(x|y)$ (source given target) is counterintuitive — it arises from Bayes' rule. Think: "how likely is the English $x$ if I assumed the French $y$ was the intended meaning?"

  

**Pipeline:** Sentence Alignment -> Word Alignment -> Phrase Extraction -> Decoding

  

**Downsides:** complex pipelines, struggles with idioms, heavy feature engineering, limited long-range context.

  

## Neural Machine Translation (NMT)

  

$$P(y \mid x) = \prod_{t=1}^{T} P(y_t \mid y_{<t}, x)$$

  

**Encoder:** reads source sentence -> produces context representation.

  

**Decoder:** generates target sentence token by token: $P(y_t \mid y_{<t}, c)$

  

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/MT-RNN.png" alt="MT-RNN" width="150">

  

A Bi-RNN encoder and an RNN decoder with teacher forcing; the bottleneck is the fixed context vector — attention solves this.

  

## Attention

  

Problem: compressing the entire sentence into **one vector** loses information.

  

$$c_t = \sum_i \alpha_{t,i} h_i$$

  

where $h_i$ = encoder states, $\alpha_{t,i}$ = attention weights.

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/MT-attn.png" alt="MT-attn" width="150">

  

|Type|Name|

|-|-|

|Additive Attention|Bahdanau|

|Multiplicative Attention|Luong|

  

**Bahdanau (additive):** score$(h_i, s_t) = v^T \tanh(W_1 h_i + W_2 s_t)$ — uses a small NN.  

**Luong (multiplicative):** score$(h_i, s_t) = h_i^T W s_t$ — faster, scales better. Transformers use scaled dot-product (Luong-style).

  

## Inference (Decoding)

  

|Method|Description|

|-|-|

|Greedy Decoding|Pick best word each step — fast but suboptimal|

|Beam Search|Keep $k$ best partial translations (typical beam size 5–10)|

|Temperature sampling|Divide logits by $T$, apply softmax, then sample|

  

**Practical tricks:** Teacher forcing, scheduled sampling, length normalisation, data augmentation (back translation, synonym replacement).

  

## Evaluation Metrics

  

|Metric|Description|

|-|-|

|BLEU|Modified n-gram precision with brevity penalty|

|Chr-F|Character n-gram F-score — better for morphologically rich languages|

|TER|Translation edit rate (minimum edits to match reference)|

|ROUGE|F-score of n-gram overlap (often for summarisation)|

|BERTScore|Cosine similarity of BERT embeddings of tokens|

  

# 5 Transformers

  

## Key Idea

  

**Transformers** are neural models for sequence modelling based **only on attention** (no recurrence or convolution), introduced in **2017**.

  

Advantages: parallel computation, handles long-range dependencies, scales well.

  

## Self-Attention

  

Each token **looks at every other token** to build its representation.

  

|Component|Role|

|-|-|

|Query (Q)|What the token is looking for|

|Key (K)|What each token offers|

|Value (V)|Information to return|

  

$$\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

  

**Steps:** (1) Compute similarity $QK^T$ -> (2) Scale by $\sqrt{d_k}$ -> (3) Softmax -> attention weights -> (4) Weighted sum of $V$.

  

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/self-attn.png" alt="MT-attn" width="150">

  

In practice: $Q = X W^Q,\; K = X W^K,\; V = X W^V$, with $W^Q,W^K,W^V \in \mathbb{R}^{d_{\text{model}}\times d_k}$.

  

**Why scale by $\sqrt{d_k}$?** When $d_k$ is large, dot products grow large in magnitude, pushing softmax into regions with very small gradients (saturation). Dividing by $\sqrt{d_k}$ keeps the variance of the dot products at ~1, stabilising training.

  

## Multi-Head Attention

  

Run **multiple attentions in parallel**, then concatenate and project.

  

Example: $D = 64$, heads $= 4$, each head dim $= 16$.

  

Benefits: capture different relationships, learn multiple contextual views. Each head operates on different linear projections of Q, K, V. Concatenated outputs are projected back to dimension $D$ via learned $W^O$.

  

## Residual Connections

  

$$\text{output} = \text{Layer}(x) + x$$

  

Enables easier gradient flow and **very deep models**.

  

## Layer Normalization

  

Normalizes activations within a token's feature vector, then applies learned parameters:

  

$$x := \frac{x-\mu}{\sigma},\quad y = \gamma x + \beta$$

  

Stabilizes training and speeds up convergence.  

**LayerNorm vs BatchNorm:** LayerNorm normalises *across the feature dimension* for each token independently; BatchNorm normalises *across the batch dimension*. Transformers use LayerNorm because it works well with variable-length sequences and small batches.

  

## Position-wise Feedforward Network

  

A small **MLP applied independently to each token**:

  

$$FFN(x) = \max(0,\ xW_1 + b_1)W_2 + b_2$$

  

Same weights for all positions. Adds **non-linear computation**.

  

## Positional Encoding

  

Transformers are position-invariant, so position is injected:

  

$$\text{input} = \text{embedding}(x) + \text{positional encoding}$$

  

Original sinusoidal encodings:

  

$$PE_{(pos,2i)} = \sin\left(\frac{pos}{10000^{2i/d}}\right) \qquad PE_{(pos,2i+1)} = \cos\left(\frac{pos}{10000^{2i/d}}\right)$$

  

- Uses sin/cos at different frequencies for different dimensions.

- Allows generalisation to longer sequences than seen in training.

- Modern models often use **learned positional embeddings** instead.

  

## Transformer Encoder Layer

  

Each layer (stacked $N$ times):

  

1. Multi-Head Self-Attention  

2. Add + LayerNorm  

3. Feedforward Network  

4. Add + LayerNorm  

  

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/transformers.png" alt="Transformer" width="150">

  

## Transformer Decoder

  

Three attention components per layer:

  

1. **Masked Self-Attention** — cannot see future tokens  

2. **Cross Attention** — attends to encoder outputs  

3. **Feedforward Network**

  

### Masked Self-Attention

  

```

         I am a student

       I ✔ ✖ ✖    ✖

      am ✔ ✔ ✖    ✖

       a ✔ ✔ ✔    ✖

 student ✔ ✔ ✔    ✔

```

  

Prevents cheating during training; ensures autoregressive generation. Masked positions receive probability 0 after softmax (implemented by adding $-\infty$ before softmax).

  

### Cross Attention

  

- **Query:** decoder state

- **Key / Value:** encoder outputs

- Attention matrix shape: $T \times S$ (target length × source length)

  

Q comes from the **decoder** (what it is generating), K and V from the **encoder** (the full source). This allows the decoder to "look up" relevant source information at each step.

  

## Practical Tricks

  

|Trick|Description|

|-|-|

|Weight tying|Decoder embedding matrix = output projection matrix|

|Learning rate schedule|Warmup steps then decay|

  

# 6 Pre-trained Models

  

## Key Idea

  

Two stages: **Pre-training** (learn general language knowledge on massive unlabeled text) -> **Fine-tuning** (adapt to a specific task).

  

## Contextual Word Representations

  

Static embeddings (e.g., word2vec) give **one vector per word** regardless of context.

  

Example: "bank" (money) vs "bank" (river) -> need different vectors.

  

**Solution:** contextual embeddings, where the vector changes depending on the sentence.

  

## ELMo

  

**ELMo (Embeddings from Language Models)** uses two RNN language models:

  

1. Forward LM (left -> right)  

2. Backward LM (right -> left)

  

```

contextual vector = combine(forward LM, backward LM)

```

  

ELMo embeddings can be **added to existing models**.  

(Not a Transformer — uses stacked biLSTMs.)

  

## Architecture Types

  

|Model Type|Context Access|Example|

|-|-|-|

|Encoder|Bidirectional|BERT|

|Decoder|Left context only|GPT|

|Encoder–Decoder|Input + output|T5|

  

- Encoder (bidirectional) -> best for understanding tasks (classification, NER, QA).  

- Decoder (left-to-right) -> best for generation.  

- Encoder-Decoder -> best for conditional generation (translation, summarisation).

  

## BERT

  

**BERT (Bidirectional Encoder Representations from Transformers):** Transformer encoder, bidirectional context, produces a vector for **every token**.

  

### Training Objectives

  

**1. Masked Language Modeling (MLM)**

  

Randomly hide 15% of tokens and predict them:

  

- 80% -> replace with `[MASK]`

- 10% -> random word

- 10% -> unchanged

  

(The 10%/10% split prevents the model from relying solely on `[MASK]` — at fine-tuning time `[MASK]` never appears, so the model must learn robust representations.)

  

**2. Next Sentence Prediction (NSP)**

  

Predict whether sentence B follows sentence A. (Removed in later models — RoBERTa showed it hurt performance.)

  

### Model Sizes

  

|Model|Layers|Hidden|Heads|Parameters|

|-|-|-|-|-|

|BERT-base|12|768|12|110M|

|BERT-large|24|1024|16|340M|

  

## Using Pre-trained Models

  

|Strategy|Description|

|-|-|

|Feature Extraction|Freeze model; use embeddings as input features|

|Fine-tuning|Add task layer and train end-to-end (most common)|

  

### Fine-tuning Task Examples

  

|Task|Method|

|-|-|

|Sentence classification|Use `[CLS]` token|

|Token labeling|Classify each token|

|Sentence pair tasks|Encode both sentences|

|Question answering|Predict answer span|

  

The `[CLS]` token aggregates sequence-level information. For QA, the model predicts a **start** and **end** position index over the tokens.

  

## BERT Family

  

|Model|Key Idea|

|-|-|

|RoBERTa|Better training, more data; dropped NSP|

|DeBERTa|Improved positional encoding|

|SpanBERT|Mask spans instead of words|

|DistilBERT|Smaller, faster BERT|

|ALBERT|Parameter sharing|

  

## Encoder–Decoder Pre-training (T5)

  

Fwindow**T5 (Text-to-Text Transfer Transformer):** everything is framed as text-in -> text-out.

  

```

translate English to German: Hello -> Hallo

```

  

Pre-training objective — **Span corruption**:

  

```

Input: Thank you <mask> me

Target: for inviting

```

  

T5's "text-to-text" framing means every task uses the same loss (cross-entropy on output tokens). The task identity is encoded in the input prefix.

  

## Instruction Training

  

Train models to follow **natural language instructions**.

  

A data sample:

  

```

Input: Summarize this article

Output: <summary>

```

  

**FLAN-T5**: trained on 1000+ tasks, improves zero-shot performance.

  

## Decoder Models (GPT)

  

Training objective: $P(w_t \mid w_1,\ldots,w_{t-1})$

  

|Model|Parameters|

|-|-|

|GPT|117M|

|GPT-2|1.5B|

|GPT-3|175B|

  

Used for: text generation, dialogue, summarisation, coding.

  

## Prompting Strategies

  

|Strategy|Description|

|-|-|

|Fine-tuning|Train on labeled task data|

|Zero-shot|Describe task in natural language|

|One-shot|Provide one example in the prompt|

|Few-shot|Provide several examples in the prompt|

  

## Chain-of-Thought Prompting

  

LLMs improve reasoning with explicit step-by-step prompting:

  

```

Let's think step by step.

```

  

The model generates intermediate reasoning before the final answer.

# POS Tagging & Parsing

  

## 1. POS Tagging (Sequence Labeling)

  

- **Definition**: Assigning grammatical categories (e.g., NOUN, VERB) to words.

- **Baseline**: Assign each word its most frequent tag; unknown words → NOUN (~90% accuracy).

- **HMM (Hidden Markov Model)**:

  - **Markov Assumption**: $P(t_i | t_{i-1})$ — tag depends only on previous tag.

  - **Independence Assumption**: $P(w_i | t_i)$ — word depends only on its own tag.

- **Viterbi Algorithm** (brief):

  1. **Initialization**: $v_1(j) = \pi_j \cdot b_j(w_1)$

  2. **Recursion**: $v_t(j) = \max_{i=1}^N [v_{t-1}(i) \cdot a_{ij}] \cdot b_j(w_t)$

  3. **Termination**: Best path probability $P^* = \max_i v_T(i)$

  (Use log probabilities to avoid underflow.)

- **Neural Approach**: RNN/LSTM with a softmax layer; trained with cross-entropy per word.

  

## 2. Constituency Parsing (Phrase Structure)

  

```

                                      S

                                    /   \

                                  NP     VP

                                /  \     |

                              Det   N    V

                              the  cat  sleeps

```

  

- **Definition**: Breaking sentences into nested hierarchical phrases (constituents) using **Context-Free Grammar (CFG)**.

- **CKY Algorithm** (brief):

  - Input: sentence $w_1 \dots w_n$, grammar in **Chomsky Normal Form (CNF)** ($A \rightarrow B C$ or $A \rightarrow w$).

  - Fill triangular table $\pi[i,j]$ with non-terminals spanning $i..j$.

  - **Base case**: if $A \rightarrow w_i$, then $A \in \pi[i,i]$.

  - **Recursive step**: for each split $k$, if $B \in \pi[i,k]$ and $C \in \pi[k+1,j]$ and $A \rightarrow B C$, then $A \in \pi[i,j]$.

  - Sentence is valid if $S \in \pi[1,n]$.

  

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/CKY.png" alt="CKY" width="150">

  

### PCFG + CKY (from slides notation)

  

A **Probabilistic CFG** is:

  

$$PCFG = (T, N, S, R, q)$$

  

- $T$ = terminals, $N$ = non-terminals, $S$ = start symbol, $R$ = rules, $q(r)$ = probability of rule $r$.

  

Parse tree probability: $P(t) = \prod_{r \in t} q(r)$.

  

Define DP table:

  

$$\pi[i,j,X] = \text{maximum probability of a constituent with non-terminal } X \text{ spanning words } i..j.$$

  

Goal: $\pi[1,n,S]$ (highest probability parse for the sentence).

  

**Base case** (lexical rules):  

$\pi[i,i,X] = \begin{cases} q(X \rightarrow w_i) & \text{if rule exists} \\ 0 & \text{otherwise} \end{cases}$

  

**Recursive case** (for spans $i..j$):  

$$\pi[i,j,X] = \max_{\substack{k,Y,Z \\ X \rightarrow YZ \in R}} \Big( q(X \rightarrow YZ) \times \pi[i,k,Y] \times \pi[k+1,j,Z] \Big)$$

  

## 3. Dependency Parsing (Relational Structure)

  

- **Definition**: Connecting words via directed binary links (arcs) representing grammatical relations.

- **Key Concepts**:

  - **Head (Governor)**: origin of the arc.

  - **Dependent (Modifier)**: destination of the arc.

  - **Projectivity**: tree is projective if no crossing arcs when words are in linear order.

- **Arc-Standard Transition-Based Parsing (MALT parser)**:

  - State: $(Stack, Buffer, Relations)$.

  - Transitions:

    1. **SHIFT**: Move word from Buffer to Stack.

    2. **LEFT-ARC**: Add relation $(S_1 \rightarrow S_2)$, pop $S_2$ (second element from stack).

    3. **RIGHT-ARC**: Add relation $(S_2 \rightarrow S_1)$, pop $S_1$ (top element from stack).

  - Decisions guided by a classifier (e.g., MLP) using features (top of stack, first in buffer, etc.).

- **Evaluation**: **UAS** (structure only) and **LAS** (structure + relation labels).

  

<img src="https://raw.githubusercontent.com/HongleiGu/Imperial-Y3-Cheatsheet/main/images/NLP/MALT.png" alt="MALT" width="150">

  

### Treebanks

Annotated corpora containing syntactic parse trees (e.g., Penn Treebank) used for training and evaluating parsers. They contain sentences annotated with POS tags, constituency trees, and dependency trees.

  

## Comparison Summary

  

| Feature | Constituency Parsing | Dependency Parsing |

|-|-|-|

| **Focus** | Phrasal groupings (NP, VP) | Word-to-word relations |

| **Nodes** | Terminal & non-terminal labels | Lexical items (words) only |

| **Complexity** | $O(n^3)$ (with CKY) | $O(n)$ to $O(n^2)$ |

| **Accuracy** | ~97% | ~95% |

## Parameter Counting

  

### Common Conventions (RNN-style models)

  

Let:

  

*   $d_x$ = input dimension (e.g., embedding size)

*   $d_h$ = hidden state dimension

*   $d_y$ = output dimension (number of classes)

  

Typical naming conventions:

  

|Symbol|Meaning|Shape|

|-|-|-|

|$U$ or $W_{xh}$|input → hidden weights|$d_h \times d_x$|

|$W$ or $W_{hh}$|hidden → hidden weights|$d_h \times d_h$|

|$V$ or $W_{hy}$|hidden → output weights|$d_y \times d_h$|

  

General rule:$  

num\_of\_parameters = rows \times columns$

  

(assuming **no bias terms**).

  

For RNNs (LSTM GRU alike), the structure is Input -> Embedding -> RNN/LSTM/GRU -> (Linear) -> Output

  

the exam question usualy tells you to ignore the embeddings and the linear layer

* * *

  

### Example 1 — Simple RNN classifier

  

Given:

  

*   embedding dimension $d_x$

*   hidden dimension $d_h$

*   output classes $d_y$

  

#### Weights

  

Hidden → hidden:$

W_{hh} \in \mathbb{R}^{d_h \times d_h}  

$

  

Input → hidden:$

W_{xh} \in \mathbb{R}^{d_h \times d_x}  

$

  

Hidden → output:$

W_{hy} \in \mathbb{R}^{d_y \times d_h}  

$

  

#### Total parameters

  

$$  

d_h^2 + d_h d_x + d_y d_h  

$$

  

* * *

  

### Example 2 — LSTM classifier

  

An LSTM contains **4 gates**:

  

*   forget gate

*   input gate

*   output gate

*   candidate cell

  

Each gate has its own **input → hidden** and **hidden → hidden** weights.

  

Per gate:$  

(d_h \times d_x) + (d_h \times d_h)  

$

  

For **4 gates**:$

4(d_h d_x + d_h^2)  

$

  

Output layer:$

W_{hy} \in \mathbb{R}^{d_y \times d_h}  

$

  

#### Total parameters

  

$$  

4(d_h d_x + d_h^2) + d_y d_h  

$$

  

* * *

  

### Example 3 — GRU

  

A GRU contains **3 gates**:

  

*   reset gate $r_t$

*   update gate $z_t$

*   candidate state $g_t$

  

Each gate has:

  

*   input → hidden weights

*   hidden → hidden weights

  

Input → hidden matrices:$  

W_{ir}, W_{iz}, W_{ig} \in \mathbb{R}^{d_h \times d_x}  

$

  

Hidden → hidden matrices:$  

W_{hr}, W_{hz}, W_{hg} \in \mathbb{R}^{d_h \times d_h}  

$

  

#### Total parameters (single direction)

  

$$  

3(d_h d_x + d_h^2)  

$$

  

#### Bidirectional GRU

  

Two independent GRUs (forward + backward):$  

2 \times 3(d_h d_x + d_h^2)  

$

  

* * *

  

### Example 4 — Transformer self-attention head

  

Let:

  

*   $d_{\text{model}}$ = model dimension

*   $d_k$ = key/query dimension

*   $d_v$ = value dimension

  

Projection matrices:

  

* $  

W^Q \in \mathbb{R}^{d_{\text{model}} \times d_k}  

$

  

* $  

W^K \in \mathbb{R}^{d_{\text{model}} \times d_k}  

$

  

* $  

W^V \in \mathbb{R}^{d_{\text{model}} \times d_v}  

$

  

#### Parameters per head

  

$$  

d_{\text{model}} d_k + d_{\text{model}} d_k + d_{\text{model}} d_v  

$$

  

If $d_k = d_v$:

  

$$  

3 d_{\text{model}} d_k  

$$

  

(Note: the output projection $W^O$ is typically counted at the **multi-head layer level**.)

  

* * *

  

### Example 5 — Transformer feedforward sublayer

  

Let:

  

*   $d_{\text{model}}$ = model dimension

*   $d_{ff}$ = inner feedforward dimension

  

First projection:$  

W_1 \in \mathbb{R}^{d_{\text{model}} \times d_{ff}}  

$

  

Second projection:$  

W_2 \in \mathbb{R}^{d_{ff} \times d_{\text{model}}}  

$

  

#### Total parameters

  

$$  

d_{\text{model}} d_{ff} + d_{ff} d_{\text{model}}  

$$

  

$$  

= 2 d_{\text{model}} d_{ff}  

$$

|Model|Gates|Total parameters (no bias)|

|-|-|-|

### Describe Training Process

#### Skip-gram/CBOW

1.  The **context words** (e.g., _cat_ and _on_) are converted into **one-hot vectors**.

2.  Each vector is multiplied by the **embedding matrix WWW** to obtain their **word embeddings**.

3.  The embeddings of the context words are **summed or averaged** to produce a **single context representation**.

4.  This representation is multiplied by the **output matrix W′W'W′** to produce scores for all words in the vocabulary.

5.  A **softmax** converts these scores into probabilities.

6.  The model predicts the **center word** (e.g., _sat_).

7.  The weights of the matrices WWW and W′W'W′ are **updated during training** to improve the prediction.

  
  

*   **Skip-gram:** center word → predict context words

*   **CBOW:** context words → predict center word (modify step 1)

  

### Other Common Question Types

  

**Perplexity interpretation**

  

> _A model achieves PPL = 10 on a vocab of 10,000. Is this good?_

  

Yes — a uniform model would have PPL = 10,000. PPL = 10 means the model behaves as if choosing between only 10 equally likely words at each step.

  

* * *

  

**Attention weight shape**

  

> _In cross-attention with source length 20 and target length 15, what is the shape of the attention matrix?_

  

$15 \times 20$ — each of the 15 target tokens attends over all 20 source tokens.

  

* * *

  

**Softmax vs sigmoid**

  

> _When would you use sigmoid vs softmax in the output layer?_

  

Sigmoid for **binary** or **multi-label** classification (each class independent). Softmax for **multi-class** classification (classes mutually exclusive, probabilities sum to 1).

  

* * *

  

**BPE tokenisation**

  

> _Why might "unrelated" tokenise differently from "related"?_

  

BPE merges are learned by frequency. "related" may appear often enough that `r-e-l-a-t-e-d` gets merged into a single token, while "unrelated" is rarer so `un` might merge but the rest stays fragmented — or the `un` prefix breaks an otherwise common merge sequence.

  

* * *

  

**Beam search**

  

> _With beam size k=1, what does beam search reduce to?_

  

Greedy decoding — it only keeps the single best hypothesis at each step.
